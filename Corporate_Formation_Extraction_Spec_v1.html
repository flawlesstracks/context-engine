How This Works for Justin's Workflow When a document is uploaded, the AI matches it to a document type below, then extracts every field listed. Each extracted value gets tagged with: the source document, page location, confidence score, and sensitivity level. Critical fields route to manual review; standard fields are eligible for Bulk Approve.

Cross-Document Validation Rules (AI should auto-flag mismatches)
Rule 1: corp.business_name = ein.business_name = f2553.corp_name = ss4.legal_name (exact match required)
Rule 2: ein.number = f2553.ein (EIN must match across all docs)
Rule 3: corp.authorized_shares = pref.total_shares (if preferred stock attachment exists)
Rule 4: corp.filing_date ≈ ss4.start_date ≈ f2553.date_incorporated (should be same or very close)
Rule 5: corp.state = f2553.state = ss4.incorporation_jurisdiction (all must be "Georgia")
Rule 6: Sum of f2553.shareholders[].ownership should = 100%

Legend

CRITICAL Never auto-approve. SSN, EIN, TIN — malpractice-risk fields.
HIGH Human review recommended. Legal names, dates, share counts.
STANDARD Eligible for Bulk Approve at ≥0.85 confidence.
LOW Informational. Auto-approve eligible.
Business Entity: Corporation, LLC, partnership
Person Entity: Officer, shareholder, agent, designee
